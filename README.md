# Web3 Research Dashboard

个人 Web3 投研工作台（V1.2 MVP + V1.3 AI 决策引擎）：单页仪表盘 + AI 投研决策辅助系统，全 mock 数据过渡中，界面语言为简体中文。

## 技术栈

- **Next.js 15**（App Router）
- **React 19**
- **Tailwind CSS 4**（`@tailwindcss/postcss`）
- **TypeScript 5**
- **Anthropic API** — 日报生成 / AI 决策辅助

## 启动命令

```bash
npm run dev        # 开发服务器
npm run build      # 生产构建
npm run lint       # ESLint
npm test           # 冒烟测试（node tests/smoke.test.mjs）
npm run report     # 生成每日 AI 投研决策报告（需 ANTHROPIC_API_KEY）
npm run report:dry # 组装报告上下文数据（无需 API key）
npx tsc --noEmit   # 类型检查
```

## 架构概述

数据与 UI 严格分层，单向依赖：

```
data/  →  lib/  →  app/  →  components/dashboard/
```

| 层级 | 目录 | 职责 |
|------|------|------|
| Mock 数据 | `data/` | 静态快照与 accessor；**仅通过** `data/index.ts` 对外暴露 |
| 业务逻辑 | `lib/` | 纯函数，无 React / fetch；决策合成、校验、展示格式化 |
| 应用壳 | `app/` | `page.tsx` 编排数据与组件；单路由 `/` |
| UI 组件 | `components/dashboard/` | 纯展示，经 props 接收数据 |

### AI 决策辅助系统（V1.3）

```
knowledge/  +  prompts/  +  live API  →  scripts/run-daily-report.mjs  →  reports/
```

| 层级 | 目录 | 职责 |
|------|------|------|
| 知识库 | `knowledge/` | 市场状态机、仓位决策框架、BTC 周期框架、产品规则、概念词典 |
| 提示词 | `prompts/` | 系统角色设定 + 每日决策报告模板 |
| 脚本 | `scripts/` | 数据获取 → 上下文组装 → Anthropic API 调用 |
| 输出 | `reports/` | 生成的每日报告（Markdown），gitignored |

## V1.2 MVP 约束

- **仅 mock 数据**：无外部 API、无数据库、无 `fetch` / `axios`
- **单页应用**：所有模块在 `/`，不新增子路由
- **观察池 ≠ 买入清单**：文案避免绝对买卖建议；允许「建议减仓」并附理由
- **首页展示范围**：观察池约 30 币，首页仅 Top 5 异动 + Top 10 Alpha，不渲染完整 30 币表
- **BTC 为周期锚点**，非单纯价格展示
- **模块顺序**（PRD 12.1）：决策 Hero → BTC 周期 → 市场环境 → 异动 Top 5 → 强信号 → Alpha → 仓位建议 → 风险 → 实际仓位对比 → 每日复盘（可折叠）→ 遗留模块（可折叠）
- 所有 V1.2 快照须共用同一 `asOf` 日期（由 `lib/data-guards.ts` 在开发环境校验）

## V1.3 AI 决策引擎

### 系统目标

1. **提高决策质量** — 每次判断基于规则框架 + 数据，不靠感觉
2. **降低失败率** — 在不利环境中主动建议防守
3. **控制回撤** — 在回撤扩大前触发三级减仓响应
4. **避免情绪交易** — 识别 FOMO、恐慌、报复交易等情绪信号
5. **提高仓位管理能力** — 根据市场状态动态调整风险暴露
6. **提高市场阶段判断准确率** — 通过状态机框架做出一致性判断

### 市场状态机（Market Regime Engine）

Agent 在每次分析中通过 9 维度 × 7 状态的判定矩阵识别当前市场：

| # | 状态 | 风险 | 核心策略 |
|---|------|------|----------|
| 1 | 强趋势牛市 | 低 | 持有 + 逢调加仓 |
| 2 | BTC 独涨 | 中低 | BTC/ETH 为主，山寨观望 |
| 3 | 山寨轮动 | 中 | 趋势跟踪，不追高 |
| 4 | 中性震荡 | 中 | 低吸高抛，控制频次 |
| 5 | 防守市场 | 中高 | 防守为主，减少操作 |
| 6 | 狂热末期 | 高 | 逐步减仓，不新增 |
| 7 | 熊市反弹 | 高 | 快进快出或不参与 |

判定维度：BTC 趋势、ETH/BTC、TOTAL3、ETF 资金流、稳定币流向、恐惧贪婪、资金费率、OI、宏观。

详见 `knowledge/market-regime-framework.md`

### 仓位决策系统（Position Sizing Engine）

四种风险模式，根据市场状态自动切换：

| 模式 | 总仓位 | 适用状态 | 核心原则 |
|------|--------|----------|----------|
| 激进 | 70-90% | 强趋势牛市、山寨轮动 | 趋势跟踪，保留现金缓冲 |
| 中性 | 40-60% | 中性震荡、BTC 独涨 | 区间操作，不押注方向 |
| 保守 | 15-30% | 防守市场、熊市反弹 | 现金为主，仅保留底仓 |
| 风险关闭 | 0-10% | 狂热末期、P0 风险 | 现金为王，等待冷静期 |

三级回撤控制：预警线（−8%）→ 行动线（−12%）→ 强制线（−15%）

详见 `knowledge/position-sizing-framework.md`

### 风险控制逻辑

- **硬止损**：总仓位 −15% 无条件转入风险关闭模式
- **单币止损**：BTC −10% / ETH −12% / 山寨 −8%~−12% / 热点 −5%
- **加仓三步检查**：市场状态 → 仓位上限 → 风险承受
- **减仓执行表**：按信号严重程度从 24h 检查到立即执行
- **48 小时冷静期**：触发强制线后强制等待
- **禁止追涨 7 条**：24h 涨幅 >15%、资金费率 >0.15%、恐惧贪婪 >85 等

### Agent 决策边界

**可以：**
- 判定市场状态（7 选 1）
- 给出仓位范围建议
- 建议减仓并附理由
- 指出不建议参与的方向
- 识别并提醒情绪交易信号

**不可以：**
- 执行交易或连接交易所
- 给出精确买卖价位
- 承诺收益或预测价格
- 违反禁止词规则
- 在未判定市场状态的情况下直接给仓位建议

### 禁止事项

- 禁止喊单、必涨、稳赚等绝对化措辞
- 禁止在狂热末期加仓
- 禁止在 200D MA 下方新增仓位（保守/中性模式）
- 禁止在资金费率 >0.15% 时追多
- 禁止使用杠杆（任何模式下）
- 禁止跳过市场状态判定直接给仓位建议
