# Web3 Research Dashboard — 路线图

## 版本演进

### V1.0：BTC 单点看板（已归档）

- BTC 价格 + 简单指标展示
- 无多资产、无 AI 决策

### V1.2：多资产观察池 MVP（已完成 · 2026-05）

**定位：** 每日投研决策型首页，所有数据为静态 mock。

- 30 币观察宇宙 + S/A/B/C/D 分层
- 首页决策卡 → BTC 周期 → 市场环境 → 异动 Top 5 → 强信号 → Alpha Top 10 → 仓位建议 → 风险提示
- 数据可信度底栏（TASK-019）
- 代币传导结构化（TASK-020）
- 链下尽调结构化（TASK-021）
- 每日复盘区（TASK-023）
- 实际仓位对比分析（TASK-024）
- 决策流程 / UI 产品结构优化（TASK-025）

**约束：** 纯 mock、无 API、单页 `/`、不自动交易、不喊单。

### V1.3：AI 决策辅助系统（已完成 · 2026-05）

**定位：** 在 V1.2 内容卡片之上增加 AI 决策引擎，实时数据驱动市场状态判定与仓位建议。

| 阶段 | 内容 | 关键产出 |
|------|------|----------|
| P0 | 市场状态机引擎 | 7 状态 × 9 维度判定矩阵，打分制分类 + 置信度 |
| P1 | 风险模式选择器 + 仓位计算器 | 4 风险模式 + 三级回撤控制 + PnL/分配计算 |
| P2 | AI 决策编排器 + 审计追踪 | 字段级 live/demo 合并 → `AiDecisionSnapshot` → 审计记录 |
| P3 | 实时数据接入 | CoinGecko（价格）+ Alternative.me（恐惧贪婪）+ DeFiLlama（稳定币）+ CoinGlass（费率/OI）；用户持仓编辑入口 + 30+ 币种自动市价填充 |
| P4-0 | 动态渲染 | `force-dynamic` 每请求实时拉取 API 数据，不再 build-time 静态生成 |

**AI 引擎卡片为实时数据；V1.2 内容卡片仍为 demo mock。** 引擎判定结果以独立 AI 决策辅助区块展示在页面中部（强信号之前）。

### P4：回测与历史复盘（规划中）

**定位：** 验证引擎判定准确率，积累历史决策记录。

- 历史数据回放：输入历史 BTC 价格 + 链上指标 → 回放引擎判断
- 准确率统计：状态判定 vs 实际走势对比
- 误判分类：假阳性（过度防守）/ 假阴性（过度乐观）
- 改进反馈：基于回测结果优化判定阈值

### 后续方向（未排期）

- 强链 / 强赛道 / 强协议对接 DefiLlama 实时数据（替换 mock）
- Alpha 池链上指标自动采集（TVL / Fees / Revenue）
- 历史观察池跟踪表 + 升级/降级规则引擎自动化
- WebSocket/SSE 实时行情推送（当前为 per-request 拉取）
- 飞书 / Notion 自动采集与推送
- 多钱包净值追踪
- 个性化风险权重配置

## 架构原则（跨版本不变）

- **单页 `/`**：不新增子路由
- **数据 → 逻辑 → UI 严格单向**
- **观察池 ≠ 买入清单**
- **AI 可建议减仓，不可喊单/承诺收益**
- **API 失败不回退整体页面，字段级兜底**
