# 市场状态机 · Market Regime Framework

> AI 投资决策系统的核心引擎。Agent 在进行任何分析前，必须先通过本框架判定当前市场状态。
> 状态判定是后续仓位建议、操作指引和风险控制的前提。

---

## 一、七种市场状态

### 概览

| # | 状态 | 风险等级 | 推荐总仓位 | 核心策略 |
|---|------|----------|------------|----------|
| 1 | 强趋势牛市 | 低 | 70-90% | 持有 + 逢调加仓 |
| 2 | BTC 独涨 | 中低 | 55-75% | BTC/ETH 为主，山寨观望 |
| 3 | 山寨轮动 | 中 | 45-70% | 趋势跟踪，不追高 |
| 4 | 中性震荡 | 中 | 40-60% | 低吸高抛，控制频次 |
| 5 | 防守市场 | 中高 | 25-40% | 防守为主，减少操作 |
| 6 | 狂热末期 | 高 | 20-35% | 逐步减仓，不新增 |
| 7 | 熊市反弹 | 高 | 15-30% | 快进快出或不参与 |

## 二、各状态详细判定

### 状态 1：强趋势牛市

**直观描述：** BTC 趋势向上 + ETH 跟随 + 山寨普涨 + 资金持续流入。最健康的做多环境。

**判断条件（需 ≥5 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格 > 200D MA，且 MA 斜率向上 | 必满足 |
| ETH | ETH/BTC 比率稳定或上升（未明显走弱） | 必满足 |
| TOTAL3 | 山寨总市值同步上升，TOTAL3/BTC 比率稳定 | 必满足 |
| ETF | BTC/ETH 现货 ETF 连续 5 日净流入 | 高 |
| 资金流 | 稳定币总市值持续增长，交易所 USDT 余额上升 | 高 |
| 情绪 | 恐惧贪婪 55-80（乐观但非极端） | 中 |
| Funding | 永续合约资金费率正常（0.01%-0.05%） | 中 |
| OI | 持仓量随价格上涨同步增长（健康杠杆） | 中 |
| 宏观 | 美联储偏鸽或中性，DXY 走弱或稳定 | 低 |

**风险等级：** 低（趋势环境友好，但需警惕过热信号）

**推荐仓位范围：**
- BTC/ETH 主流：50-65%
- 稳定币/现金：10-20%
- Alpha/山寨：15-25%
- 高风险热点：3-5%

**适合做什么：**
- BTC：持有为主，回调 5-8% 可加仓
- 山寨：趋势跟踪，强者恒强，做轮动不做补涨
- Meme：仅小仓位（≤2%），快进快出
- 观望：不适用，趋势环境应保持参与
- 高频：不推荐——趋势环境减少交易频率，拿住比交易重要
- 防守：不适用

**禁止行为：**
- 禁止频繁止盈（趋势中卖飞比套牢更常见）
- 禁止重仓追高单币（即使看好，单币不超过总仓位 15%）
- 禁止因"涨高了"就空仓（仓位管理≠清仓）
- 禁止忽视 OI 和资金费率（过热信号出现时需警惕）

---

### 状态 2：BTC 独涨

**直观描述：** BTC 在涨但山寨不跟，ETH/BTC 走弱。资金集中在 BTC，山寨流动性被抽血。

**判断条件（需 ≥4 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格趋势向上或横盘偏强 | 必满足 |
| ETH | ETH/BTC 比率持续下降（≥7 天） | 必满足 |
| TOTAL3 | TOTAL3/BTC 比率下降，山寨总市值横盘或下跌 | 必满足 |
| ETF | BTC ETF 净流入，ETH ETF 持平或流出 | 高 |
| 资金流 | 稳定币流入集中在 BTC 交易对，山寨交易量萎缩 | 高 |
| 情绪 | BTC 恐惧贪婪 > 60，但山寨社区情绪低迷 | 中 |
| Funding | BTC 资金费率略高（>0.05%），山寨资金费率为负 | 中 |
| OI | BTC OI 增长，山寨 OI 下降 | 中 |
| 宏观 | 无明显风险事件 | 低 |

**风险等级：** 中低（BTC 方向正确，但山寨风险高）

**推荐仓位范围：**
- BTC/ETH 主流：40-55%
- 稳定币/现金：25-35%
- Alpha/山寨：8-15%
- 高风险热点：0-3%

**适合做什么：**
- BTC：持有，回调可小幅加仓
- 山寨：以观察为主，只关注相对 BTC 未走弱的强势山寨
- Meme：禁止——BTC 独涨期 Meme 流动性最差
- 观望：对山寨保持观望，等待 ETH/BTC 企稳信号
- 高频：不推荐
- 防守：对山寨板块执行防守策略

**禁止行为：**
- 禁止在 BTC 独涨时重仓山寨（"BTC 涨完山寨补涨"不是必然规律）
- 禁止在 ETH/BTC 跌破关键支撑时逆势抄底 ETH
- 禁止忽视 BTC 独涨可能演变为"BTC 独涨→BTC 回调→山寨暴跌"

---

### 状态 3：山寨轮动

**直观描述：** BTC 稳定或缓慢上涨，ETH 和山寨开始跑赢 BTC。资金从 BTC 溢出到山寨。

**判断条件（需 ≥4 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格稳定或缓慢上涨，BTC Dominance 下降 | 必满足 |
| ETH | ETH/BTC 比率上升（≥5 天） | 必满足 |
| TOTAL3 | 山寨总市值上升，TOTAL3/BTC 比率上升 | 必满足 |
| ETF | BTC ETF 流入放缓但不明显流出；ETH ETF 可能开始流入 | 高 |
| 资金流 | 交易所山寨交易量占比上升，稳定币流入山寨交易对 | 高 |
| 情绪 | 山寨社区活跃度上升，"Altseason"讨论增多 | 中 |
| Funding | 热门山寨资金费率开始升高（>0.1% 需警惕） | 中 |
| OI | 山寨 OI 开始增长 | 中 |
| 宏观 | 风险偏好上升，DXY 走弱 | 低 |

**风险等级：** 中（轮动机会多，但切换快、假突破多）

**推荐仓位范围：**
- BTC/ETH 主流：30-45%
- 稳定币/现金：20-30%
- Alpha/山寨：20-35%
- 高风险热点：3-8%

**适合做什么：**
- BTC：保留底仓，不追
- 山寨：趋势跟踪，强者恒强。关注率先突破的赛道龙头
- Meme：小仓位（≤5%）参与热度最高的 1-2 个
- 观望：不适合——轮动期应保持参与
- 高频：可适度提高交易频率，但需严格止损
- 防守：不适用

**禁止行为：**
- 禁止轮动中频繁换仓追涨（容易被左右打脸）
- 禁止重仓某一个赛道（轮动意味着板块切换快）
- 禁止在资金费率 >0.1% 时加仓（过热信号）
- 禁止忽视"山寨轮动→狂热末期"的转换信号：OI 飙升 + 费率极端 + 情绪 >85

---

### 状态 4：中性震荡

**直观描述：** BTC 在区间内震荡，无明显方向。山寨分化严重，部分涨部分跌。

**判断条件（需 ≥4 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格在 200D MA 附近 ±15% 区间震荡 ≥2 周 | 必满足 |
| ETH | ETH/BTC 横盘，无明显方向 | 必满足 |
| TOTAL3 | 山寨总市值横盘，无趋势性变化 | 必满足 |
| ETF | ETF 流入流出交替，净变化接近零 | 高 |
| 资金流 | 稳定币市值横盘，无明显流入流出 | 高 |
| 情绪 | 恐惧贪婪 40-60 | 中 |
| Funding | 资金费率接近零（中性） | 中 |
| OI | OI 横盘，无趋势 | 中 |
| 宏观 | 政策等待期，无明确信号 | 低 |

**风险等级：** 中（方向不明，假突破频繁）

**推荐仓位范围：**
- BTC/ETH 主流：30-45%
- 稳定币/现金：35-50%
- Alpha/山寨：10-18%
- 高风险热点：0-3%

**适合做什么：**
- BTC：低吸高抛，区间上下沿操作。区间突破前不追
- 山寨：仅关注有独立催化剂的项目，不做纯 Beta 交易
- Meme：不推荐——震荡市 Meme 流动性最差
- 观望：对无催化剂的山寨选择观望
- 高频：不推荐——震荡市反复止损
- 防守：保持较高现金比例，等待方向确认

**禁止行为：**
- 禁止在区间中间位置开仓（上下沿操作，中间不动）
- 禁止对突破方向预判重仓（等确认后再行动）
- 禁止因"无聊"而频繁交易（震荡市交易成本最高）
- 禁止忽视突破时成交量的确认（无量突破大概率假突破）

### 状态 5：防守市场

**直观描述：** BTC 趋势向下或关键支撑失守。资金流出风险资产，情绪转向恐慌。

**判断条件（需 ≥4 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格跌破 200D MA，且 MA 斜率转平或向下 | 必满足 |
| ETH | ETH/BTC 可能上升或下降（防守期 ETH 表现不一） | — |
| TOTAL3 | 山寨总市值下降，跌幅通常大于 BTC | 必满足 |
| ETF | BTC/ETH ETF 连续净流出 | 高 |
| 资金流 | 稳定币从交易所流出（资金离场），USDT 溢价消失 | 高 |
| 情绪 | 恐惧贪婪 < 35，社交媒体恐慌情绪上升 | 中 |
| Funding | 资金费率转为负（空头主导） | 中 |
| OI | OI 下降（杠杆出清中） | 中 |
| 宏观 | 美联储鹰派、DXY 走强、风险事件（战争/监管/系统性风险） | 高 |

**风险等级：** 中高（趋势不利，主要目标是保住本金）

**推荐仓位范围：**
- BTC/ETH 主流：20-35%
- 稳定币/现金：50-60%
- Alpha/山寨：5-10%
- 高风险热点：0%

**适合做什么：**
- BTC：仅保留底仓。定投可小幅进行但降低频率
- 山寨：仅持有基本面最强且跌幅已大的 A 级项目。不在下跌中加仓
- Meme：绝对禁止
- 观望：以观望为主。等待 BTC 重新站上 200D MA
- 高频：不推荐
- 防守：提高稳定币占比，减少操作频率

**禁止行为：**
- 禁止"抄底"——下跌趋势中抄底是最常见的亏损原因
- 禁止因"已经跌了很多"就加仓——便宜可以更便宜
- 禁止在 BTC 未站稳关键支撑前买入山寨
- 禁止忽视宏观风险事件（加息/监管/系统性风险）而"技术分析抄底"

---

### 状态 6：狂热末期

**直观描述：** 市场情绪极端乐观，价格加速冲高，杠杆率飙升。所有指标显示过热，但散户仍在跑步进场。

**判断条件（需 ≥5 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 价格加速上涨（斜率陡峭），或出现高位长上影/穿头破脚 | 必满足 |
| ETH | ETH 和山寨也在暴涨，"万币齐飞" | 必满足 |
| TOTAL3 | 山寨总市值抛物线式上升 | 必满足 |
| ETF | ETF 流入可能仍在持续但开始出现单日净流出 | 高 |
| 资金流 | 稳定币流入速度跟不上价格上涨（杠杆推动而非现货推动） | 高 |
| 情绪 | 恐惧贪婪 > 85 且持续 ≥7 天 | 必满足 |
| Funding | 主流山寨资金费率 > 0.1%（极端时 >0.3%） | 必满足 |
| OI | OI 创历史新高，且增长速度远超价格涨幅 | 必满足 |
| 宏观 | 通常发生在降息预期最强时，但市场已充分定价 | 低 |

**风险等级：** 高（随时可能急跌。主力正在出货，散户在接盘）

**推荐仓位范围：**
- BTC/ETH 主流：20-30%
- 稳定币/现金：40-55%
- Alpha/山寨：5-15%
- 高风险热点：0-5%（仅利润部分，本金不参与）

**适合做什么：**
- BTC：分步减仓，先减山寨，后减 BTC。减仓节奏：每涨 10% 减 20%
- 山寨：逐步清仓高估值山寨，利润落袋。仅保留最强基本面
- Meme：仅用利润参与，本金绝对不进
- 观望：对大部分标的转向观望
- 高频：不推荐——狂热期波动剧烈，容易双向爆仓
- 防守：逐步转入防守模式

**禁止行为：**
- 禁止"我就再拿一天"——狂热末期的顶部通常发生在 1-2 天内
- 禁止新增仓位——此时的买入短期胜率极低
- 禁止在资金费率 >0.2% 时做多——持仓成本极高
- 禁止因 FOMO 而追高——"别人都在赚"是最危险的交易信号
- 禁止全仓杠杆——狂热末期一个 10% 的回调就能爆仓

### 状态 7：熊市反弹

**直观描述：** 长期下跌趋势中的短期反弹。幅度可观（15-30%）但持续性差，随后通常创新低。

**判断条件（需 ≥4 项满足）：**

| 维度 | 条件 | 权重 |
|------|------|------|
| BTC | 从低点反弹 15-30%，但仍在 200D MA 下方 | 必满足 |
| ETH | 跟随反弹，但 ETH/BTC 未趋势性走强 | 必满足 |
| TOTAL3 | 山寨跟随反弹，幅度通常大于 BTC（超跌反弹特征） | 必满足 |
| ETF | ETF 短暂流入但量不大，可能很快转为流出 | 高 |
| 资金流 | 反弹期间有资金流入，但量级明显弱于牛市 | 高 |
| 情绪 | 恐惧贪婪从 <20 反弹至 30-50，但很快回落 | 中 |
| Funding | 资金费率短暂转正后可能快速转负 | 中 |
| OI | OI 小幅反弹但远低于前期高点 | 中 |
| 宏观 | 可能有利好催化（如降息预期），但大趋势仍偏紧 | 低 |

**风险等级：** 高（熊市反弹是最容易套人的陷阱。看起来像反转，实际是延续下跌前的诱多）

**推荐仓位范围：**
- BTC/ETH 主流：10-25%
- 稳定币/现金：60-75%
- Alpha/山寨：3-8%（仅最强基本面）
- 高风险热点：0%

**适合做什么：**
- BTC：小幅参与或不参与。如果参与，设好止损（-5% 即离场）
- 山寨：仅持有 A 级项目底仓。反弹中不加仓
- Meme：绝对禁止
- 观望：最优策略——不参与熊市反弹
- 高频：不推荐——熊市反弹的假突破率最高
- 防守：保持防守。反弹不是反转

**禁止行为：**
- 禁止把反弹当成反转——"这次不一样"是熊市最贵的四个字
- 禁止在反弹中重仓——熊市反弹后通常创新低
- 禁止因反弹就调整长期仓位计划——反弹是噪音，趋势是信号
- 禁止在 BTC 没有重新站上 200D MA 前加仓

## 三、状态转换信号

### 转换图

```
强趋势牛市 ←→ 山寨轮动
    ↓              ↓
  BTC 独涨      狂热末期
    ↓              ↓
  中性震荡 ←→ 防守市场
    ↓              ↓
  防守市场 ←→ 熊市反弹
    ↑_______________|
```

### 关键转换信号

| 从 → 到 | 核心信号 |
|----------|----------|
| 强趋势牛市 → BTC 独涨 | ETH/BTC 连续 7 日走弱 + TOTAL3/BTC 开始下降 |
| 强趋势牛市 → 狂热末期 | OI 创历史新高 + 资金费率极端 + 恐惧贪婪 >85 |
| BTC 独涨 → 山寨轮动 | ETH/BTC 企稳回升 + 山寨交易量开始放大 |
| BTC 独涨 → 中性震荡 | BTC 突破失败 + 区间震荡 ≥2 周 |
| 山寨轮动 → 狂热末期 | 情绪 >85 + 资金费率 >0.15% + OI 飙升 |
| 山寨轮动 → 中性震荡 | 轮动速度加快（一日游）→ 热点散乱 → 震荡 |
| 中性震荡 → 防守市场 | BTC 跌破区间下沿 + 200D MA 失守 |
| 中性震荡 → 强趋势牛市 | BTC 放量突破区间上沿 + 200D MA 斜率转正 |
| 防守市场 → 熊市反弹 | 超跌反弹 15-30%，但 200D MA 仍在下方 |
| 防守市场 → 中性震荡 | BTC 重新站上 200D MA + ETF 流出停止 |
| 狂热末期 → 防守市场 | BTC 单日跌幅 >8% + OI 暴跌（杠杆出清） |
| 熊市反弹 → 防守市场 | 反弹失败 + BTC 跌破反弹起点（创新低） |

## 四、状态判定优先级

当多指标冲突时，按以下优先级判定：

1. **BTC 与 200D MA 的关系**（最基础的趋势方向）— 权重 30%
2. **ETH/BTC 与 TOTAL3/BTC 的走势**（山寨相对强度）— 权重 25%
3. **ETF 资金流 + 稳定币流向**（机构+聪明钱方向）— 权重 20%
4. **OI + 资金费率**（杠杆热度，反转信号）— 权重 15%
5. **恐惧贪婪 + 宏观**（情绪+外部环境）— 权重 10%

> 原则：**趋势 > 资金 > 杠杆 > 情绪**

## 五、分析输出规范

Agent 每次分析市场状态时，必须输出：

1. **当前状态**：七选一
2. **判断依据**：列出命中了哪些判断条件（具体数值）
3. **信心程度**：高（≥6 项条件满足）/ 中（4-5 项）/ 低（<4 项或指标冲突）
4. **风险等级**：当前状态对应的风险等级
5. **距离下一状态的信号**：最近的可能转换方向 + 需要观察的指标
6. **仓位倾向**：当前状态下推荐的总仓位范围
7. **核心操作建议**：2-3 句话
