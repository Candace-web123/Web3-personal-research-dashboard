# 仓位决策系统 · Position Sizing Framework

> AI 投资决策系统的仓位管理引擎。Agent 在判定市场状态后，必须通过本框架确定具体仓位等级、风险暴露和操作边界。
> 核心原则：**仓位管理决定了 80% 的投资结果。选币只是杠杆，仓位才是本金。**

---

## 一、四种风险模式

### 概览

| 模式 | 适用市场状态 | 总仓位范围 | 最大单币占比 | 杠杆 |
|------|-------------|-----------|-------------|------|
| 保守 | 防守市场、熊市反弹 | 15-30% | 8% | 0 |
| 中性 | 中性震荡、BTC 独涨 | 40-60% | 12% | 0 |
| 激进 | 强趋势牛市、山寨轮动 | 70-90% | 18% | 0 |
| 风险关闭 | 狂热末期、P0 风险事件 | 0-10% | 3% | 0 |

> **硬约束：任何模式下均不使用杠杆。** 这个系统服务于保守型个人投资者，本金有限，杠杆=清零风险。

---

## 二、各模式详细规则

### 模式 1：保守模式（Conservative）

**触发条件（任一满足即启用）：**
- 市场状态判定为「防守市场」或「熊市反弹」
- BTC 价格低于 200D MA 且 MA 斜率向下
- 恐惧贪婪 < 30 且持续 ≥3 天
- ETF 连续 5 日净流出
- P0 级风险事件发生

**仓位结构：**

| 资产类别 | 占比 | 说明 |
|----------|------|------|
| BTC | 10-20% | 仅保留底仓，不对冲不操作 |
| ETH | 5-10% | 跟随 BTC |
| 稳定币/现金 | 60-75% | 主力仓位。放在有收益的稳定币协议（如 Aave） |
| Alpha/山寨 | 3-8% | 仅持有 A 级项目。不在下跌中加仓 |
| 高风险 | 0% | 绝对不持有 |

**加仓条件（需全部满足）：**
1. BTC 重新站上 200D MA 且站稳 ≥3 天
2. 恐惧贪婪回升至 40+
3. ETF 流出停止或转为流入
4. 无新增 P0/P1 风险

**减仓条件（任一触发即执行）：**
1. BTC 跌破前低
2. P0 风险事件发生
3. 持仓浮亏 >10%（总仓位层面，非单币）
4. 稳定币收益率发生系统性风险（如 USDC 脱锚）

**回撤控制：**
- 总仓位回撤硬止损：**−10%**（触及后无条件降至 10% 仓位）
- 单币回撤硬止损：**−15%**（触及后清仓该币）

**禁止行为：**
- 禁止因"跌了很多应该见底了"而抄底
- 禁止在 200D MA 下方新增任何仓位
- 禁止因无聊而在震荡中频繁交易

---

### 模式 2：中性模式（Neutral）

**触发条件（任一满足即启用）：**
- 市场状态判定为「中性震荡」或「BTC 独涨」
- BTC 在 200D MA 附近震荡，方向不明
- 恐惧贪婪 40-60，无极端情绪
- 恐惧贪婪 30-60 且持续

**仓位结构：**

| 资产类别 | 占比 | 说明 |
|----------|------|------|
| BTC | 25-35% | 核心仓位。区间操作 |
| ETH | 10-18% | 与 BTC 保持合理比例 |
| 稳定币/现金 | 35-50% | 中等现金缓冲 |
| Alpha/山寨 | 10-18% | 仅关注有独立催化剂的项目 |
| 高风险 | 0-3% | 几乎不碰 |

**加仓条件（需 ≥3 项满足）：**
1. BTC 放量突破震荡区间上沿
2. ETH/BTC 走强
3. ETF 转为持续净流入
4. 稳定币市值上升
5. 市场状态转为「强趋势牛市」

**减仓条件（任一触发即执行）：**
1. BTC 跌破震荡区间下沿
2. ETF 持续流出 ≥3 天
3. 单币跌幅 >12%（破坏技术结构）
4. 出现 P1 级以上风险

**回撤控制：**
- 总仓位回撤预警线：**−8%**（触及后审视，考虑降仓位）
- 总仓位回撤硬止损：**−12%**（触及后无条件降至 30% 仓位）
- 单币回撤硬止损：**−12%**

**禁止行为：**
- 禁止在区间中间位置频繁开平仓（交易成本吃掉利润）
- 禁止因"震荡太久该突破了"而提前押注方向
- 禁止总交易频率超过每周 3 次（震荡市越交易越亏）

---

### 模式 3：激进模式（Aggressive）

**触发条件（需全部满足）：**
- 市场状态判定为「强趋势牛市」或「山寨轮动」
- BTC 价格 > 200D MA 且 MA 斜率向上
- ETF 净流入（至少不是持续流出）
- 恐惧贪婪 50-80（乐观但非极端）
- 资金费率未极端（主流币 <0.1%）

**仓位结构：**

| 资产类别 | 占比 | 说明 |
|----------|------|------|
| BTC | 30-40% | 最大仓位。趋势持有，不做短线 |
| ETH | 15-25% | 趋势跟随 |
| 稳定币/现金 | 10-20% | 最小现金缓冲 |
| Alpha/山寨 | 15-25% | 趋势跟踪强势山寨 |
| 高风险 | 3-8% | 仅用利润参与 |

**加仓条件（需 ≥3 项满足）：**
1. 趋势延续，BTC 回调 5-8% 不破关键均线
2. ETF 流入持续
3. 稳定币市值继续增长
4. 山寨轮动健康（TOTAL3/BTC 上升，非一日游）
5. 资金费率正常（未极端）

**减仓条件（任一触发即执行）：**
1. BTC 单日跌幅 >6% 且跌破 20D MA
2. 恐惧贪婪突破 85
3. 资金费率 >0.15% 且持续
4. OI 出现异常飙升
5. ETH/BTC 连续 5 日走弱
6. ETF 出现单日大额净流出

**回撤控制：**
- 总仓位回撤预警线：**−10%**（触及后降低山寨占比 50%）
- 总仓位回撤硬止损：**−15%**（触及后无条件转中性模式）
- 单币回撤硬止损：**−10%**（趋势中更紧的止损）

**禁止行为：**
- 禁止"趋势确认后就满仓"——即使激进模式也保留 10-20% 现金
- 禁止追高已涨超 30%（7 日内）的标的
- 禁止单币超过总仓位 18%
- 禁止因"这次不一样"而忽略减仓信号

---

### 模式 4：风险关闭模式（Risk-Off）

**触发条件（任一满足即启用）：**
- 市场状态判定为「狂热末期」
- P0 级风险事件（协议被攻击、系统性风险、交易所风险）
- 恐惧贪婪 > 90 且 OI 创新高
- BTC 单日跌幅 >10%
- 个人总仓位回撤达到 −15%

**仓位结构：**

| 资产类别 | 占比 | 说明 |
|----------|------|------|
| BTC | 0-8% | 仅保留极少量或不持有 |
| ETH | 0-5% | 跟随 BTC |
| 稳定币/现金 | 75-100% | 现金为王 |
| Alpha/山寨 | 0% | 全部清仓 |
| 高风险 | 0% | 绝对不持有 |

**恢复条件（需全部满足，不可跳过）：**
1. 触发条件已消除
2. 市场状态转为非高风险（中性震荡或更优）
3. 恐惧贪婪回落至 60 以下
4. 稳定币收益率环境安全
5. 至少等待 48 小时冷静期

**禁止行为：**
- 禁止在风险关闭模式下交易任何山寨
- 禁止因"跌了很多"就提前结束风险关闭
- 禁止因无聊而在风险关闭期做短线
- 禁止跳过 48 小时冷静期

## 三、加仓与减仓通用规则

### 加仓三步检查

在考虑增加任何仓位前，Agent 必须完成以下三步检查：

```
第一步：市场状态检查
  → 当前是什么市场状态？
  → 这个状态允许加仓吗？
  → 当前是否接近状态转换边界？

第二步：仓位上限检查
  → 当前总仓位是否已达模式上限？
  → 目标币种是否已达单币上限？
  → 加仓后是否会超过任何一个上限？

第三步：风险检查
  → 加仓后的最大回撤是否可承受？
  → 未来 24-48h 是否有已知催化/风险事件？
  → 资金费率是否合理？
  → 如果判断错误，止损位在哪？
```

### 减仓执行规则

| 触发信号 | 减仓幅度 | 执行时限 |
|----------|----------|----------|
| 市场状态降级（如强趋势→BTC 独涨） | 降山寨 50% | 24h 内 |
| BTC 跌破 200D MA | 降总仓位至 30% 以下 | 48h 内 |
| 恐惧贪婪 >85 + 资金费率极端 | 降山寨 70% | 当天 |
| P0 风险事件 | 降至风险关闭模式 | 立即 |
| 总仓位回撤触及硬止损 | 按硬止损规则执行 | 立即 |
| 单币回撤触及硬止损 | 清仓该币 | 立即 |

### 禁止追涨条件

以下情况**绝对禁止**新增仓位：

1. **任何资产 24h 涨幅 >15%** — 不追
2. **任何资产 7 日涨幅 >50%** — 不追（等回调 15-20% 后再评估）
3. **资金费率 >0.15%** — 不追（持仓成本过高）
4. **恐惧贪婪 >85** — 不追（情绪极端，反转概率高）
5. **成交量异常放量（超过 30 日均量 3 倍）** — 不追（可能是顶部放量出货）
6. **社交媒体集中讨论某币** — 不追（FOMO 信号）
7. **自己没有做过基本面研究** — 不追（追了自己不懂的币=赌博）

## 四、回撤控制体系

### 三级回撤响应

```
Level 1 — 预警线（总仓位 −8%）
  → 暂停新增仓位
  → 审视所有持仓，检查是否有异常
  → 不强制减仓，但不再加仓

Level 2 — 行动线（总仓位 −12%）
  → 减仓山寨板块 50%
  → 降低总仓位至模式下限
  → 仅保留 BTC/ETH 底仓

Level 3 — 强制线（总仓位 −15%）
  → 转入风险关闭模式
  → 清仓所有山寨
  → BTC/ETH 仅保留最多 10%
  → 至少 48 小时冷静期
```

### 单币回撤止损

| 资产类别 | 止损线 | 说明 |
|----------|--------|------|
| BTC | −10% | 跌破关键均线止损 |
| ETH | −12% | 略宽于 BTC |
| Alpha（A/B 级） | −12% | 基本面强的可略宽 |
| Alpha（C/D 级） | −8% | 低评级严格止损 |
| 高风险热点 | −5% | 快进快出，绝不扛单 |

> 止损后冷静期：清仓一个币种后，至少 24 小时不重新买入同一币种。

## 五、情绪纪律

### Agent 必须识别的情绪交易信号

当用户出现以下表述时，Agent 必须主动提醒风险：

| 用户表述 | 情绪类型 | Agent 响应 |
|----------|----------|-----------|
| "别人都在赚" | FOMO | 提醒：这是最危险的交易信号。回归仓位规则。 |
| "这次不一样" | 过度自信 | 提醒：这次通常和上次一样。检查周期位置。 |
| "已经跌了这么多" | 锚定效应 | 提醒：便宜可以更便宜。检查趋势是否改变。 |
| "先买一点试试" | 冲动交易 | 提醒：不确定时等待比交易更好。 |
| "再拿一天就走" | 贪婪 | 提醒：检查是否已触及减仓条件。 |
| "止损出局算了" | 恐慌 | 提醒：检查是触及止损还是情绪反应。 |
| "我一定会赚回来" | 报复交易 | 提醒：这是回撤后最危险的心态。冷静 48h。 |

### Agent 强制冷静规则

以下情况 Agent 必须输出"建议冷静，暂不操作"：

1. 用户在同一天内第二次要求分析交易机会
2. 用户提到的币种过去 24h 涨跌幅 >20%
3. 用户在总仓位已触及预警线时要求加仓
4. 用户要求分析 3 个以上新币种（信息过载→冲动决策）
5. 用户使用"赶快""赶紧""马上"等紧迫性词汇

## 六、分析输出规范

Agent 每次给出仓位建议时，必须输出：

1. **当前风险模式**：保守/中性/激进/风险关闭
2. **建议总仓位**：范围 + 依据
3. **建议仓位结构**：BTC/ETH/稳定币/山寨/高风险的百分比分布
4. **止损位**：总仓位止损线 + 关键单币止损位
5. **下次审视时机**：什么条件下应该重新评估仓位
6. **当前最大风险**：1 句话
7. **情绪检查**：是否存在需要警惕的情绪信号
